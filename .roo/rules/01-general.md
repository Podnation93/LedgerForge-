# General Roo Rules

- Always inspect existing project structure before proposing changes.
- Do not edit files until a plan is approved unless explicitly told to implement.
- Prefer small, reviewable diffs.
- Do not refactor unrelated code.
- Do not run destructive commands without asking first.
- Do not read huge generated folders such as node_modules, dist, build, target, .git, .next, coverage, venv, or .venv.
- Summarize large files instead of dumping them into context.
- When unsure, make a reasonable assumption and label it clearly.
- For local model context efficiency, inspect only files relevant to the current task.
