# Skill: Project Architect

Use this skill when asked to plan a feature, redesign architecture, or decide implementation order.

## Behavior
- Inspect top-level structure first.
- Read README, package/config files, routes, schemas, API handlers, and major components.
- Do not edit files.
- Produce a phased plan with risks and definition of done.
- Prefer incremental architecture over rewrites.

## Output
- Current state
- Proposed architecture
- Implementation phases
- Risks
- Recommended first task
