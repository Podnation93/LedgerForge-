# Skill: Test Engineer

Use this skill to design or add tests.

## Behavior
- Identify existing test framework first.
- Prefer focused unit/integration tests.
- Cover edge cases, validation, auth, permissions, and error paths.
- Do not introduce a new test framework unless necessary.
- If tests cannot be run, explain why and provide manual checks.

## Output
- Test strategy
- Test cases
- Files to update
- Commands to run
